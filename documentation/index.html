Prozen – Next.js Business & Consulting Template

Clean, modern, animated website template built with Next.js (App Router) and tailored micro‑interactions.

Next.js App Router SSR/SSG

1. Overview

Prozen Next.js Template is a modern template designed for business, finance, corporate and consulting websites. It focuses on:

  • Three hero homepages (Main, Corporate, Insurance)
  • Next.js App Router – file-based routing with layouts
  • Smooth scrolling powered by Lenis
  • Scroll animations handled by AOS (Animate On Scroll)
  • Reusable UI sections for services, projects, team, FAQ, pricing, blog, and more

2. Getting Started

2.1. Requirements

  • Node.js 18+ (recommended 20+)
  • npm 9+ or pnpm / yarn (optional)
  • A modern browser (Chrome, Edge, Firefox, Safari)

2.2. Installation

  1. Install dependencies

    cd prozen-next
npm install

  2. Run development server

    npm run dev

    By default Next.js will start on http://localhost:3000.

  3. Build for production

    npm run build
npm start

2.3. Project Highlights

  • File-based routing with the Next.js app/ directory
  • Context‑based Offcanvas menu for mobile navigation
  • Lenis for buttery‑smooth scroll and keyboard scroll
  • AOS for one‑time scroll‑in animations on route changes
  • Swiper for hero sliders and testimonials
  • react-hook-form for accessible contact forms

3. Folder Structure

Below is a condensed view of the project structure. Only the most important folders are shown. 

prozen-next/
├─ package.json              # Scripts and dependencies
├─ next.config.*             # Next.js configuration (if present)
├─ public/                   # Static assets served at site root
│  └─ images/                # Template images (update these with your own)
└─ app/                      # Next.js App Router
   ├─ layout.js              # Root layout (global CSS, metadata, Providers)
   ├─ providers.jsx          # Global client providers (Lenis, Offcanvas, AOSInit, UI)
   ├─ style/
   │  ├─ style.css           # Theme styles
   │  └─ lenis.css           # Lenis-specific overrides
   ├─ (pages)/               # Route group (keeps URLs clean)
   │  ├─ home/               # /home (if used)
   │  ├─ home-two/           # /home-two
   │  ├─ home-three/         # /home-three
   │  ├─ about-us/           # /about-us
   │  ├─ service/            # /service
   │  ├─ service-details/    # /service-details
   │  ├─ project/            # /project
   │  ├─ project-details/    # /project-details
   │  ├─ team/               # /team
   │  ├─ team-details/       # /team-details
   │  ├─ pricing/            # /pricing
   │  ├─ faq/                # /faq
   │  ├─ blog/               # /blog
   │  ├─ blog-list/          # /blog-list
   │  ├─ blog-details/       # /blog-details
   │  └─ contact/            # /contact
   ├─ components/
   │  ├─ common/             # Header, Offcanvas, etc.
   │  ├─ ui/                 # Cursor, Preloader, BackToTop, Search, ScrollToTop, AOSInit
   │  └─ pages/              # Page sections grouped by page type
   ├─ contexts/              # Context providers (client-only)
   ├─ hooks/                 # Client hooks (count-up, sticky header, forms)
   └─ data/                  # Centralized content (text, counters, image paths)

The goal is to keep routing handled by the Next.js App Router, page composition in `app/(pages)/*/page.jsx`, and reusable sections in `app/components/pages/*`. Content (titles, copy, images) lives in the `app/data/*` files so buyers can update text without touching components.

6. Layout & Routing

6.1. App Router entry – app/layout.js

In Next.js (App Router) you don’t bootstrap the app with a custom entry file. Instead, global CSS and wrappers live in app/layout.js. This template imports global CSS there and wraps the app with Providers.

6.2. Global providers – app/providers.jsx

app/providers.jsx wraps the application with client-side providers and global UI that should exist on every route:

  • LenisProvider – smooth scroll & keyboard handling
  • OffcanvasProvider – global mobile menu state
  • AOSInit – initializes and refreshes AOS on route changes

Global UI that should be visible on every page – Offcanvas menu, ScrollToTop, BackToTop, custom Cursor – is rendered inside Providers so it persists across navigation.

7. Pages & Sections

7.1. Homepages

  • Home (`/`) – Business / consulting default homepage with hero, services, about, stats, projects, process, team, testimonials, blog.
  • Home Two (`/home-two`) – Corporate layout with alternate hero, stats, service carousel, and FAQ testimonials.
  • Home Three (`/home-three`) – Insurance‑focused layout with hero video card, services, process, pricing, testimonials and FAQ.

7.2. Inner pages

  • About Us – Mission, process, stats, team, testimonials.
  • Services & Service Details
  • Projects & Project Details
  • Team & Team Details
  • Pricing – Pricing tiers + FAQ accordion.
  • Blog / Blog List / Blog Details
  • FAQ – Custom accordion component (no Bootstrap JS).
  • Contact – Google Maps area + contact form (react-hook-form).

Each route page in app/(pages)/*/page.jsx is a thin wrapper that composes sections from app/components/pages/* and uses data from app/data/*.

7.3. Routing in Next.js (file-based)

Next.js App Router uses file-based routing. Create a folder with a page.jsx to define a route:

  • app/page.jsx → /
  • app/(pages)/about-us/page.jsx → /about-us
  • app/(pages)/blog-details/page.jsx → /blog-details

8. Animations & Effects

8.1. Scroll animations (AOS)

AOS is initialized in AOSInit.jsx and refreshed on route changes using usePathname. Components opt into animations using data-aos attributes: 

<div
  data-aos="fade-up"
  data-aos-delay="300"
>
  ...
</div>

8.2. Smooth scrolling (Lenis)

LenisContext.jsx creates a single Lenis instance and wires it to a requestAnimationFrame loop and keyboard events. The custom ScrollToTop component integrates with Lenis to keep scroll positions in sync on every route change.

8.3. Sliders (Swiper)

Swiper is used for hero sliders and testimonials. Each instance imports only the required modules (e.g. Navigation, Autoplay) to keep bundle size small.

8.4. Micro‑interactions

  • Offcanvas mobile menu with animated submenus
  • Search overlay with smooth slide‑down animation
  • Back‑to‑top button that fades in after scroll
  • Custom cursor following mouse with hover states
  • Count‑up stats using useCountUp + IntersectionObserver

9. Customization

9.1. Colors & typography

Theme colors, typography and spacing are controlled in app/style/style.css using CSS variables (e.g. --ht-theme-color, --ht-heading-color). Updating these variables will update the entire template.

9.2. Content

All text, counters, and image paths live in app/data/*:

  • homeData.js – main homepage hero, services, projects, posts.
  • homeTwoData.js, HomeThreeData.js – alternate layouts.
  • aboutUsData.js, faqData.js, blogData.js, etc.

9.3. Adding a new section

  1. Create a new component under app/components/pages/<group>/ (e.g. services, home).
  2. Wire its content via a new or existing data file.
  3. Include the component in the proper route app/(pages)/*/page.jsx.

9.4. Images (Next.js)

Store your images in public/images/ and reference them with absolute paths like /images/.... The template uses next/image for optimized images (keep width and height props for correct aspect ratio).

9.5. Logo & navigation

  • Logo: replace files in public/images/logo/ and update the logo path in header components under app/components/common/ or app/components/pages/*/.
  • Menus: update menu data in app/data/headerMenuData.js (and related menu data files).

9.6. Metadata / SEO

Global SEO is configured in app/layout.js using Next.js metadata (for example: metadataBase, openGraph, and twitter). Update those values to match your brand and domain.

9.7. Client vs Server Components

In App Router, files are Server Components by default. Add "use client" only when you need hooks, browser APIs, or client-only libraries (for example: Swiper sliders, Lenis scrolling, AOS initialization, or DOM event listeners).

10. Performance Notes

  • Built on Next.js – optimized production builds and modern rendering.
  • Only necessary Swiper modules are imported per slider.
  • Animations are lightweight (CSS + AOS), no heavy JS animation engines.
  • Lenis smooth scroll is created once and shared via context.

For further optimization, keep client components small and prefer Server Components for static sections. Next.js automatically code-splits by route.

11. Frequently Asked Questions

11.1. Does this template use jQuery?

No. All interactivity is implemented with Next.js client components and vanilla JS.

11.2. Can I use only one of the homepages?

Yes. Remove the unused section imports from the relevant route file under app/(pages)/ and update navigation items in the header/offcanvas menu data files.

11.3. How do I change images?

Update the image paths in the data files (e.g. homeData.js, aboutUsData.js) and place your assets under the public images/ directory.

11.4. How do I change the primary color?

Edit the CSS variable --ht-theme-color inside app/style/style.css.
